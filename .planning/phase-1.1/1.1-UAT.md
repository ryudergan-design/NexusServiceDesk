# Teste de Aceitação do Usuário (UAT): Fase 1.1 (Refinamento e Segurança)

## Objetivo
Validar a transição bem-sucedida para o banco de dados real (SQLite), a segurança do fluxo de autenticação com criptografia e a usabilidade em dispositivos móveis.

---

### Cenário 1: Autenticação de Administrador Inicial (Seed)
**Objetivo:** Verificar se o acesso administrativo via seed está funcional.
- **Passos:**
    1. Acessar a página de `/login`.
    2. Inserir o e-mail: `jefrsonsales@outlook.com`.
    3. Inserir a senha definida no script de seed.
    4. Clicar em "Entrar".
- **Resultado Esperado:** Redirecionamento bem-sucedido para o Dashboard com perfil de administrador.

### Cenário 2: Cadastro de Novo Usuário (Fluxo de Aprovação)
**Objetivo:** Validar se novos cadastros são criados como "Pendentes" por padrão.
- **Passos:**
    1. Acessar a página de `/register`.
    2. Preencher os dados de perfil (Nome, E-mail, Senha).
    3. Concluir o registro.
    4. Tentar fazer login imediatamente com os novos dados.
- **Resultado Esperado:** O login deve falhar com a mensagem "Aguardando aprovação do administrador" (ou similar), e os dados devem estar persistidos no `dev.db`.

### Cenário 3: Segurança de Credenciais (Criptografia)
**Objetivo:** Garantir que senhas não sejam armazenadas em texto plano.
- **Passos:**
    1. Abrir o banco de dados `dev.db` (via Prisma Studio ou SQLite Browser).
    2. Consultar a tabela `User`.
    3. Observar o campo `password`.
- **Resultado Esperado:** A senha deve estar em formato de hash (ex: `$2b$10$...`), tornando-a ilegível.

### Cenário 4: Navegação Responsiva (Mobile)
**Objetivo:** Validar a usabilidade em telas pequenas.
- **Passos:**
    1. Acessar o Dashboard em um dispositivo móvel ou simulador de navegador.
    2. Verificar se a Sidebar lateral está oculta.
    3. Clicar no ícone de "Menu Hambúrguer" no Header.
    4. Selecionar uma opção no menu lateral que surgiu (Drawer).
- **Resultado Esperado:** O menu deve abrir suavemente, ser legível e fechar ao selecionar uma rota ou clicar fora.

### Cenário 5: Persistência de Sessão e Logout
**Objetivo:** Validar o ciclo completo de sessão.
- **Passos:**
    1. Realizar login com sucesso.
    2. Atualizar a página (F5).
    3. Clicar no botão de Logout no User Menu.
    4. Tentar acessar `/dashboard` diretamente pela URL.
- **Resultado Esperado:** A sessão deve persistir após o refresh e, após o logout, o acesso direto ao dashboard deve ser bloqueado e redirecionado para o login.
